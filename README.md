## Про проєкт

Практична реалізація методики автоматизованого тестування на базі **ROS 2 Jazzy**. Система автоматично перевіряє працездатність коду маніпулятора **PUMA 560** після кожної зміни через CI/CD пайплайн.

## Структура репозиторію

* **.github/workflows/ci.yml** — налаштування GitHub Actions (Jazzy/Ubuntu 24.04).
* **puma_testing/** — логіка тестування (pytest):
    * `test_talker.py` — юніт-тести вузла (6 тестів).
    * `test_puma_system.py` — інтеграційні тести моделі (15 тестів).
* **puma560_description/** — об'єкт тестування (URDF модель, 3D меші, Launch файли).

## Рівні тестування

### 1. Юніт-тестування (6 тестів)

Перевірка вузла `MinimalPublisher`: ініціалізація, коректність імені, стабільність публікації, вміст повідомлень та негативне тестування (обробка помилкових вхідних даних).

### 2. Інтеграційне тестування (15 тестів)

* **Статична верифікація:** Валідність XML-моделі (xacro), наявність 7 STL-файлів геометрії та конфігурацій RViz2.
* **Динамічна верифікація:** Контроль топіка `/joint_states` (формат 6 суглобів) та перевірка фізичних лімітів кутів (±2π).

## Інструкція з запуску

Система розрахована на **ROS 2 Jazzy** та **Ubuntu 24.04**.

\`\`\`bash
source /opt/ros/jazzy/setup.bash
colcon build --packages-select puma_testing
source install/setup.bash
colcon test --packages-select puma_testing --event-handlers console_direct+
colcon test-result --verbose
\`\`\`
